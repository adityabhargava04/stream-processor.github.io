Stream Processor

GPU-Accelerated Fluid Simulation — CS 184 Project Proposal

Team: Gracelynn Quek, Ruiyun Wang, Julian Vale, Aditya Bhargava

Summary

Stream Processor is a GPU-accelerated fluid simulation system focused on rendering atmospheric effects like rain and fog. We're using a hybrid PIC/FLIP particle-grid approach combined with custom shader-based rendering to produce physically convincing, visually rich results. CUDA parallelism lets us scale to particle counts and scene complexities that would otherwise be too slow to run in real time.

Problem Description

Atmospheric effects like rain and fog are everywhere in games, simulations, and visual effects - and they're notoriously hard to get right. The core challenge isn't just making water "look wet"; it's simulating how thousands (or millions) of particles interact with each other, with boundaries, and with light, all while keeping the whole thing running fast enough to be useful.

The scale of these simulations is what makes them tricky. A single rain scene might involve tens of thousands of particles moving under gravity, wind, and turbulence, all while needing to respond to scene geometry. Grid-based solvers are fast but tend to smear out fine detail. Pure particle methods preserve detail well but get expensive quickly. And on top of the physics, you still need the rendering pipeline - shaders, lighting, transparency - to make everything look convincing rather than just technically correct.

Our plan is to tackle this with a hybrid PIC/FLIP method, which gets the best of both worlds: the structure of a grid for efficient computation, and particles to preserve the fluid's detail and energy. We'll implement this in C++ with CUDA to push the heavy lifting onto the GPU, and write custom GLSL shaders to handle the visual side - procedural textures, depth-based blending, and light attenuation for atmospheric depth.

Goals and Deliverables

What We Plan to Deliver

Part 1 - Fluid Simulation via PIC/FLIP

We'll build a hybrid particle-grid simulation framework from scratch in C++/CUDA. The core loop involves transferring particle velocities to a grid, solving for pressure on the grid, then transferring updated velocities back to the particles - the standard PIC/FLIP pipeline. GPU parallelism handles the particle-to-grid transfers and pressure solve efficiently at scale. The simulation should produce stable, artifact-free fluid motion and basic atmospheric flow behavior over long durations.

Part 2 - Texturing and Shader Rendering

  • Procedural texturing with depth-based blending and transparency for fog/rain appearance
  • Basic light attenuation and scene depth cues for realism
  • Point sprite rendering for particles as a baseline visual

Deliverables: A collection of rendered images showing stable PIC/FLIP simulation behavior across different scenarios (rain, fog, fluid flow), plus a short performance report covering FPS at varying particle counts, volume preservation over time, and stability observations.

Evaluation Metrics:

  • Volume preservation of the simulated fluid over time (targeting minimal loss)
  • FPS under varying simulation sizes
  • Visual smoothness and absence of artifacts
  • Stability over long simulation runs

What We Hope to Deliver

If things go well, we want to push the project into interactive territory - a real-time demo where you can tweak parameters like particle count, wind force, and turbulence intensity and watch the simulation respond live.

Stretch goals on the rendering side:

  • Volumetric fog via ray-marching
  • Refraction and environment mapping for water surfaces
  • Motion blur for rain particles

Schedule

Week Focus Tasks
Week 1 Setup & Core Data Structures Configure repo, build system, and CUDA environment. Implement grid data structure and particle initialization. Begin particle-to-grid (P2G) velocity transfer.
Week 2 Full PIC/FLIP Pipeline Complete FLIP velocity update and grid-to-particle (G2P) transfer. Implement GPU pressure projection solver. Render particles as basic point sprites.
Week 3 Boundaries, Stability & Shaders Add boundary conditions and track volume preservation. Build shader pipeline - depth-based blending, transparency, basic lighting. Run performance profiles (FPS vs. particle count).
Week 4 Polish, Testing & Writeup Finish rendering pass - procedural textures, fog layering. Run long-duration stability tests. Compile rendered images, performance graphs, and written report.
